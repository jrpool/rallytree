RallyTree introduction

Trees

Do you need to operate on a tree of Rally work items?

To do that manually, you must

  1. expand each work item in the Rally web interface, until the entire tree is expanded
  2. perform a bulk edit (if possible) or an item-by-item count

Trees of Rally work items can have hundreds of nodes. One operation can take about 15 minutes. It is also vulnerable to error.

RallyTree automates that process. You specify the root user story of the tree and what you want done. Then RallyTree performs the operation on the tree.

Operations

RallyTree can perform these operations:

  • Documentation: representing a tree of user stories as JSON
  • Test-result acquisition: returning a tally of test-case verdicts and defects
  • Ownership change: ensuring all user stories and tasks have a specified owner
  • Task creation: creating tasks with specified names for each test-level user story
  • Test-case creation: creating a test case for each test-level user story
  • Test-case result creation: creating a passing result for each test case that has no result
  • Copy creation: copying a tree of user stories

RallyTree introduction

Trees

Do you need to operate on a tree of Rally work items?

To do that manually, you must

  1. expand each work item in the Rally web interface, until the entire tree is expanded
  2. perform a bulk edit (if possible) or an item-by-item count

Trees of Rally work items can have hundreds of nodes. One operation can take about 15 minutes. It is also vulnerable to error.

RallyTree automates that process. You specify the root user story of the tree and what you want done. Then RallyTree performs the operation on the tree.

Operations

RallyTree can perform these operations:

  • Copy creation: copying a tree of user stories and optionally setting copy properties
  • Scoring: reporting test-case verdicts and defects, and a score
  • Ownership change: giving all user stories, tasks, and test cases a specified owner
  • Project change: ensuring all user stories, tasks, and test cases belong to a specified project and optionally giving schedulable user stories a specified release and/or iteration
  • Schedule-state change: giving schedulable user stories a schedule state
  • Task creation: creating tasks with specified names for each leaf user story
  • Test-case creation: creating test cases for user stories
  • Pass creation: creating passing results for test cases
  • Planify: creating a test plan mirroring a tree of user stories and test cases
  • Documentation: representing a tree as JSON

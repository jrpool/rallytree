RallyTree introduction

Trees

Do you need to operate on a tree of Rally work items?

To do that manually, you must expand each work item in the Rally web interface, until the entire tree is expanded, and then perform a bulk edit. That can take you several minutes. It is also vulnerable to error.

RallyTree automates that process. You specify the root user story of the tree and what you want done. Then RallyTree performs the operation on that user story and all of its descendant user stories and tasks.

Problems? Please send questions or comments about RallyTree to Jonathan Pool.

Ownership change

One operation is to change the owner. You specify the new owner. Then RallyTree examines all the work items in the tree. Wherever the specified owner is not yet the owner, RallyTree makes the change.

Problems? Please send questions or comments about RallyTree to Jonathan Pool.

Test-case creation

Another operation is to create test cases. If you request that, RallyTree examines each test-level user story in the tree. If it has a test-case creation task but does not have a test case fulfilling that task, RallyTree creates that test case.
